 LIVE MODE: Zero Hardcoded Data Policy (Enforced Immediately)
🔁 1. Global Environment Gate
In your root config or .env:

env
Copy
Edit
MODE=LIVE
In code:

ts
Copy
Edit
const isLive = process.env.MODE === 'LIVE';
🔨 2. Purge ALL Hardcoded Defaults – Replace With Safe Nulls
🔧 Example Fix: Metrics
tsx
Copy
Edit
const display = (value: any) => (isLive ? (value ?? '--') : demoValue);

// Bad ❌
const metric = isLive ? "100%" : "98%";

// Good ✅
const metric = isLive ? fetchLiveMetric() ?? '--' : '98%';
🔧 Example Fix: Tables
tsx
Copy
Edit
const tableRows = isLive ? fetchedRows : demoRows;

if (isLive && !fetchedRows.length) return null;
📛 3. Mask All Demo Names / Labels / Titles
Scrub all instances of:

"Client A", "Sample Company", "John Smith", "100%", "42 Calls", "This Quarter"...

Hardcode blocklist search terms to grep:

bash
Copy
Edit
grep -r 'Client A' src/
grep -r '"100%' src/
grep -r '42' src/
Then kill those lines or replace with live values only:

tsx
Copy
Edit
const clientName = isLive ? realClientName : null;
🛑 4. No Fallback Logic Allowed in Live Mode
tsx
Copy
Edit
// ❌ BAD
const value = isLive ? fetchLive() : demoValue;

// ✅ GOOD
const value = isLive ? fetchLive() ?? '--' : demoValue;
